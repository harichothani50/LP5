
Need ONE process responsible for:

* coordination
* decision making
* synchronization
* resource management

This process is called:

# LEADER / COORDINATOR

---

Real distributed systems need:

* centralized decision maker
* synchronization controller
* task manager

# REAL WORLD EXAMPLE

everyone talks simultaneously.

Same in distributed systems.

# WHAT IF LEADER CRASHES?

BIGGEST DISTRIBUTED SYSTEM PROBLEM.

Suppose:
leader process dies.

System must:

* detect failure
* choose new leader

This is called:

# LEADER ELECTION

---

# 2. WHAT IS BULLY ALGORITHM?

Bully Algorithm is:
leader election algorithm where:

# Highest ID process becomes leader.

---

# WHY NAME "BULLY"?

Because:
higher ID process dominates lower ID processes.

It “bullies” smaller IDs.

---

# BASIC RULE

| Process ID | Priority        |
| ---------- | --------------- |
| Higher ID  | Higher Priority |
| Lower ID   | Lower Priority  |

---

# EXAMPLE

Processes:

```text id="x6r0pn"
P1 P2 P3 P4 P5
```

Highest ID:

```text id="mb1v7d"
P5
```

So:
P5 becomes leader.

---

# 3. WHAT HAPPENS IF LEADER FAILS?

Suppose:
P5 crashes.

Then:
remaining processes start election.

Highest active process wins.

Example:

```text id="v9q4se"
P4
```

becomes leader.

---

# IMPORTANT DISTRIBUTED SYSTEM CONCEPT

This algorithm assumes:

# Higher ID process is more powerful/reliable.

# ADVANTAGES OF BULLY ALGORITHM

| Advantage     | Reason                       |
| ------------- | ---------------------------- |
| Simple        | Easy implementation          |
| Fast election | Highest process quickly wins |
| Deterministic | Highest ID always leader     |

# DISADVANTAGES

| Disadvantage              | Reason                 |
| ------------------------- | ---------------------- |
| High message overhead     | Many election messages |
| Not scalable              | Heavy communication    |
| Assumes process IDs known | Static system needed   |

| --------------------- | ------------------------ |
