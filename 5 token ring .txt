# 1. FIRST UNDERSTAND THE MAIN PROBLEM

Suppose multiple processes want to access:

* printer
* database
* bank account
* shared file
* CPU resource

at SAME TIME.

This creates problems.

---

# REAL LIFE EXAMPLE

Imagine:

3 people trying to use:

* one bathroom
* one ATM
* one printer

If all enter simultaneously:
CHAOS.

Need controlled access.

---

# THIS IS CALLED:

# MUTUAL EXCLUSION

Meaning:

ONLY ONE process allowed inside critical resource at a time.

---

# 2. WHAT IS CRITICAL SECTION?

MOST IMPORTANT CONCEPT.

---

# CRITICAL SECTION

Part of program where:
shared resource is accessed.

Example:

* updating database
* writing file
* using printer
* modifying shared memory

---

# WHY IS CRITICAL SECTION DANGEROUS?

If two processes access simultaneously:

Data corruption occurs.

---

# EXAMPLE

Suppose:
Bank balance = 1000

Process A withdraws 500.
Process B withdraws 300.

Without synchronization:
wrong final balance may occur.

---

# THIS IS CALLED:

# RACE CONDITION

---

# WHAT IS RACE CONDITION?

When multiple processes:

* access shared resource simultaneously
* causing unpredictable behavior

---

# SOLUTION → DISTRIBUTED MUTUAL EXCLUSION

Need mechanism ensuring:
ONLY ONE process enters critical section.

---

# 3. WHAT IS TOKEN RING ALGORITHM?

One special object called:

# TOKEN

circulates among processes.

ONLY process holding token:
can enter critical section.

---

# SIMPLE RULE

| Has Token? | Can Enter CS? |
| ---------- | ------------- |
| Yes        | Allowed       |
| No         | Not Allowed   |

---

# REAL LIFE EXAMPLE

Imagine classroom talking stick.

Only student holding stick:
can speak.

Token = permission.

---

# 4. WHAT IS RING TOPOLOGY?

Processes connected in circular form.

Example:

```text id="e0p3gs"
P1 → P2 → P3 → P1
```

Token moves continuously in ring.

---

# WHY RING USED?

Simple token passing mechanism.

Each process knows:

* previous process
* next process

Easy coordination.

# WHY TOKEN RING IS IMPORTANT?

Guarantees:

* no starvation
* fairness
* mutual exclusion

---

# WHAT IS STARVATION?

When process waits forever.

Token ring avoids this because token eventually arrives.

---

# ADVANTAGES OF TOKEN RING

| Advantage     | Reason                        |
| ------------- | ----------------------------- |
| Fairness      | Token circulates sequentially |
| No starvation | Every process gets token      |
| Simple logic  | Easy implementation           |
| Deterministic | Predictable behavior          |

---

# DISADVANTAGES

| Disadvantage            | Reason                     |
| ----------------------- | -------------------------- |
| Token loss problem      | If token lost system stops |
| Slow for many processes | Token must travel          |
| Single token dependency | Failure affects system     |

---

# IMPORTANT ADVANCED CONCEPT

# TOKEN LOSS

Suppose token disappears due to crash.

Then:
NO process can enter critical section.

Real systems use:

* token regeneration
* failure detection

---

# ANOTHER ADVANCED CONCEPT

# DEADLOCK

---

# WHAT IS DEADLOCK?

Processes waiting forever.

Token ring avoids traditional deadlock because:
only one token exists.

Distributed mutual exclusion algorithm where token circulates among processes.

To guarantee only one process enters critical section.

A process never getting chance to execute.
