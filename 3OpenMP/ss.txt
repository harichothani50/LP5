# 1. FIRST UNDERSTAND THE MAIN PROBLEM

Suppose you have:

```text id="5dzj4t"
1 2 3 4 5 6 7 8
```

Need to calculate sum.

Normally:
one CPU executes sequentially:

```text id="u9xxe1"
1+2+3+4+5+6+7+8
```

This is called:

# Sequential Computing

Only ONE execution flow.

---

# PROBLEM WITH SEQUENTIAL EXECUTION

For small arrays:
fine.

But imagine:

* 10 billion elements
* scientific simulations
* AI matrix operations
* weather forecasting

One thread becomes VERY slow.

---

# SOLUTION → PARALLEL COMPUTING

Instead of ONE worker:

Use MULTIPLE workers.

Example:

| Thread   | Work |
| -------- | ---- |
| Thread 0 | 1+2  |
| Thread 1 | 3+4  |
| Thread 2 | 5+6  |
| Thread 3 | 7+8  |

Then combine results.

This is:

# Parallel Processing

---

# 2. WHAT IS OPENMP?

OpenMP =
Open Multi-Processing

It is an API/library for:

* parallel programming
* multithreading
* shared memory multiprocessing

Mostly used in:

* C
* C++
* Fortran

---

# WHY OPENMP EXISTS?

Without OpenMP:
creating threads manually is HARD.

Need:

* thread creation
* synchronization
* scheduling
* locking

OpenMP simplifies this using compiler directives.

---

# MOST IMPORTANT IDEA

You simply write:

```c id="j4l1z0"
#pragma omp parallel
```

and compiler automatically creates threads.

---

# 3. WHAT IS A THREAD?

VERY IMPORTANT CONCEPT.

---

# PROCESS VS THREAD

---

# PROCESS

A running program.

Example:

* Chrome
* VS Code
* Game

Each has separate memory.

---

# THREAD

A lightweight execution unit inside process.

Threads share:

* same memory
* same variables
* same resources

---

# REAL LIFE EXAMPLE

Restaurant:

PROCESS = Restaurant

THREADS =

* waiter
* chef
* cashier

All work together.

---

# YOUR PROGRAM USES MULTITHREADING

Multiple threads:

* compute simultaneously
* divide workload
* improve speed

# CRITICAL SECTION

```c id="jlwm7t"
#pragma omp critical
```

EXTREMELY IMPORTANT.

---

# WHY CRITICAL SECTION IS NEEDED?

All threads share:

```c id="jlwm4v"
total_sum
```

If multiple threads update simultaneously:

WRONG ANSWERS occur.

---

# EXAMPLE OF PROBLEM

Suppose:

```text id="jlwm1v"
total_sum = 10
```

Thread A adds 5.
Thread B adds 7.

Without synchronization:
final result may become:

```text id="jlwm6q"
15
```

instead of:

```text id="jlwm9w"
22
```

because updates overlap.

---

# CRITICAL SECTION SOLUTION

Only ONE thread allowed at a time.

Like bathroom lock system.

---

# PARALLELISM VS CONCURRENCY

VERY IMPORTANT VIVA QUESTION.

---

# CONCURRENCY

Tasks progress together.

Not necessarily same time.

Example:
switching between tasks rapidly.

---

# PARALLELISM

Tasks execute literally simultaneously.

Needs multiple cores.

# PRACTICAL/VIVA QUESTIONS AND ANSWERS

---

# Q1. What is OpenMP?

OpenMP is an API for parallel programming in C/C++/Fortran using multithreading in shared memory systems.

---

# Q2. What does `#pragma omp parallel` do?

It creates multiple threads and executes the following block in parallel.

---

# Q3. What is thread?

A thread is a lightweight execution unit inside a process.

---

# Q4. Difference between process and thread?

Process has separate memory.
Threads share same memory.

---

# Q5. What is race condition?

When multiple threads simultaneously modify shared data causing unpredictable results.

---

# Q6. Why critical section used?

To ensure only one thread accesses shared resource at a time.

---

# Q7. What happens if critical section removed?

Incorrect total sum may occur due to race condition.

---

# Q8. What is chunking?

Dividing workload among threads.

---

# Q9. What does `omp_get_thread_num()` return?

Current executing thread ID.

---

# Q10. Why use local_sum?

To avoid race condition during partial calculations.

---

# Q11. What is shared memory?

Memory accessible by all threads.

---

# Q12. Why OpenMP is faster?

Because multiple CPU cores work simultaneously.

---

# Q13. What is synchronization?

Controlling thread access to shared resources safely.

---

# Q14. Why `-fopenmp` needed?

Enables OpenMP support during compilation.

---

# Q15. Sequential vs Parallel Computing?

Sequential uses one execution flow.
Parallel uses multiple simultaneous execution flows.
