# 1. FIRST UNDERSTAND THE MAIN PROBLEM

Suppose you have:

| Machine  | Time     |
| -------- | -------- |
| Client 1 | 10:00:05 |
| Client 2 | 10:00:09 |
| Client 3 | 10:00:02 |
| Server   | 10:00:07 |

All machines have DIFFERENT times.

---

# WHY IS THIS A BIG PROBLEM?

In distributed systems:
machines must coordinate.

Example:

* bank transaction timestamps
* online gaming events
* distributed databases
* stock market systems
* cloud servers

If times differ:
massive errors happen.

---

# REAL EXAMPLE

Suppose:
Bank Server says:

```text id="zcixh9"
Transaction completed at 10:00:05
```

Client clock says:

```text id="ol0jtl"
Request sent at 10:00:07
```

Impossible situation:
response came BEFORE request.

This causes:

* inconsistency
* corruption
* synchronization issues

---

# SOLUTION → CLOCK SYNCHRONIZATION

Need all systems to agree on approximately same time.

---

# 2. WHAT IS BERKELEY ALGORITHM?

One central server:

* collects times from clients
* calculates average time
* sends adjustments to clients

Clients then adjust clocks.

---

# IMPORTANT

Berkeley Algorithm DOES NOT use UTC/internet time directly.

Instead:
system internally synchronizes all machines.

---

# REAL LIFE EXAMPLE

Teacher asks students:

```text id="mx7c9v"
What time do you have?
```

Students answer:

* 10:02
* 10:04
* 10:03

Teacher averages:

```text id="j9e45s"
10:03
```

Then tells everyone:

> "Adjust your clock accordingly."

That is EXACTLY Berkeley Algorithm.
